# Optional Benchmark Extension

The main experiment should remain synthetic EHR because it is deterministic, controllable, and fully reproducible without credentialed medical data. A benchmark dataset can be added as a small external validation section, not as the main result.

## Recommendation

Use this order:

1. **Main result:** synthetic longitudinal EHR generated by this repo.
2. **External support:** MedAlign small-scale evaluation if time allows.
3. **Background-only alternative:** LongBench, only to show general long-context relevance.
4. **Not recommended for this code path:** EHRSHOT, because it is structured clinical prediction rather than long-text QA retention.

## Why MedAlign Is the Best Fit

MedAlign is a clinician-generated dataset for instruction following with electronic medical records. Public materials describe it as containing longitudinal EHRs, clinician-curated instructions, clinical reasoning, timeline understanding, and multi-document synthesis.

This matches our project because our method is about:

```text
long patient context
→ context retention/compression
→ question answering or instruction answering
→ quality-efficiency trade-off
```

How to use it in the report:

```text
We use synthetic longitudinal EHR as the primary benchmark because it allows controlled evidence labels and reproducible retention analysis. We additionally discuss MedAlign as an external longitudinal EHR benchmark that could validate the same retention interface on clinician-generated instructions.
```

## Why EHRSHOT Is Less Suitable Here

EHRSHOT is a strong longitudinal EHR benchmark, but it focuses on structured EHR foundation model evaluation and clinical prediction tasks. That is useful for EHR representation learning, but it does not directly match our current long-text QA pipeline.

Use it only if the project later changes toward:

```text
structured EHR event prediction
few-shot clinical classification
AUROC / AUPRC evaluation
```

Do not use it as the main benchmark for selective text retention unless we add a new task adapter.

## Why LongBench Is Only a Background Supplement

LongBench is useful for showing that context compression is a recognized long-context problem. However, it is not healthcare-specific and not longitudinal EHR-specific.

Use it in related work or discussion, not as the core experiment.

## Reasonable Project Integration

Add benchmark support as a separate adapter path:

```text
data/benchmark/
scripts/prepare_benchmark.py
scripts/run_benchmark_experiment.py
```

Keep the same internal sample schema:

```json
{
  "sample_id": "...",
  "patient_id": "...",
  "task_type": "benchmark_instruction",
  "question": "...",
  "answer": "...",
  "gold_evidence_event_ids": []
}
```

For MedAlign, if note-level IDs or evidence IDs are available, map them into `gold_evidence_event_ids`. If not, report only QA quality and token/latency/memory, and do not report evidence recall for that benchmark.

## Minimal Implementation Plan

Do not mix benchmark files with the synthetic main pipeline. Add a separate optional command sequence:

```bash
python3 scripts/prepare_benchmark.py --input data/benchmark/medalign.jsonl --out data/processed/benchmark_qa.jsonl

python3 scripts/run_experiment.py --config configs/default.yaml --qa data/processed/benchmark_qa.jsonl --timelines data/processed/benchmark_timelines.jsonl --strategy full_context --out data/results/benchmark_full_context.jsonl

python3 scripts/run_experiment.py --config configs/default.yaml --qa data/processed/benchmark_qa.jsonl --timelines data/processed/benchmark_timelines.jsonl --strategy selective --retention-ratio 0.5 --out data/results/benchmark_selective.jsonl
```

This keeps the benchmark optional and avoids breaking the deterministic synthetic reproduction.

## What To Claim

Safe claim:

```text
Our main controlled experiment uses synthetic longitudinal EHR data. Public longitudinal EHR benchmarks such as MedAlign provide a natural next-step validation setting, but the final project focuses on reproducible controlled evidence retention.
```

Avoid claiming:

```text
The method is clinically validated on real EHR.
```

unless the benchmark is actually downloaded, processed, and evaluated.
